# Kiosk Runtime Checklist (Windows + Browser)

## Device baseline

- [ ] Windows mini-PC connected to front display.
- [ ] Auto-login enabled for kiosk service account.
- [ ] Power plan set to never sleep during operating hours.
- [ ] Screen saver and lock screen timeout disabled.

## Browser startup

- [ ] Chrome or Edge installed and updated.
- [ ] Startup target set to display app URL/local host page.
- [ ] Fullscreen or kiosk argument configured on launch.
- [ ] Browser set to restore tab if app reload is needed.

## Reliability controls

- [ ] Daily reboot schedule configured (off-hours).
- [ ] App auto-reload configured at startup.
- [ ] Network reconnect tested after router reset.
- [ ] Fallback page or local copy available.

## Security and privacy

- [ ] Disable access to browser settings for public users.
- [ ] No patient-identifiable information shown publicly.
- [ ] Feedback handling restricted to authorized staff process.
- [ ] Consent text visible before feedback submission.

## Front desk handover

- [ ] Quick restart instruction card printed.
- [ ] Contact person assigned for content updates.
- [ ] Contact person assigned for feedback triage.
